PadSpan HA v0.3.15 flow-fix notes

This build specifically targets:
- "Config flow could not be loaded: 500 Internal Server Error"
- Sidebar registration compatibility on newer HA builds

Install:
1) Stop Home Assistant
2) Remove:
   - /config/custom_components/padspan
   - /config/custom_components/padspan_ha
3) Copy /custom_components/padspan_ha from this package to /config/custom_components/
4) Start Home Assistant
5) Hard refresh browser (Ctrl+F5)
6) Try:
   - Integrations -> Add -> PadSpan HA
   - Existing PadSpan HA -> Configure (options)

If 500 persists, clear unfinished flows:
- Delete stale entries from /config/.storage/core.config_entries that reference domain "padspan" (legacy)
- Restart HA
- Retry
